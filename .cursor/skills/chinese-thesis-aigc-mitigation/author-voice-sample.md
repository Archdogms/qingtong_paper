# 笔调参考：导航与模仿要点（请先读本文，再按需读扩展）

## 与 `SKILL.md` 的关系（重要）

五篇 `skill参考论文` 原文中，部分课程作业带有**口语化连接**或 **pre 讲稿体**。使用本 skill 时，Agent 须以 **`SKILL.md`「语体底线：书面化、严谨性优先」** 为准：**论文学输出应书面、严谨**；参考文只借**结构与材料组织**，不把「口水话」当模仿目标。

**同一稿不要连跑两轮「全文降 AIGC」**：见 `SKILL.md` 章节「**同一稿件多次套用本 skill 的风险**」——第二轮易叠套话、反升 AI 率；二轮须有**新材料**或只做**局部**改红字。

## 原文来源索引（`skill参考论文/`）

- `期末论文.docx` → 设计类论述：**问题—案例（首钢大跳台）—技术路径—多学科条件**，可学**长句因果链与工科案例嵌套**；落笔时改为书面衔接（「据此」「可见」「上述结果表明」等）。  
- `长文终稿.docx` → 史论：**设问引入—史料—（1）（2）机制归纳—分派观点—辩证收束**；可学**证据密度与层次**；避免照搬口语反问密度过高的段落结构到不宜体裁。  
- `乡村振兴…docx` → 政策与案例：**政策依据—分论点—静宁苹果、黄文秀等例证—结论**；可学**例证与论述的配合**；引政策须核对用户当次是否仍需该表述。  
- `意大利台地园…docx` → 史论/读解：**小标题 + 编号分述 + 术语（柱式、台地等）**；与**正式说明文体**最接近，可优先对齐其**条理性与术语性**。  
- `pre讲稿.docx` → **口头报告**：中英夹杂、分段口语。**禁止**作为学位论文/设计说明正文的语体模板；仅可参考「如何把复杂概念分块讲清」的思路，落笔须改写为书面语。

## 建议从五篇中「抽取」的维度（非口语）

1. **篇章**：由现象或问题导入 → 本文工作界定 → 分节论证 → 局限与展望。  
2. **证据**：地名、工程名、数据、人名与文献交织的写法（须与当次课题材料一致）。  
3. **结构**：小标题、（1）（2）、「首先/其次/综上」等**书面分层**（控制密度，避免机械「第一第二」连打）。  
4. **术语**：学科名词一致、少混用口语近义词。  
5. **收束**：「一分为二」「从上述分析可知」类**学理化小结**，避免情绪化、口号化。

## 模仿禁区

- **勿**把 extended / txt **整句复制**到待提交稿。  
- **勿**为降 AIGC 而堆「其实、不得不说、大有来头」等口语标记；若保留转折，用「然而」「因此」「在此基础上」等书面形式为主。  
- **勿**把 pre 讲稿体、演讲感叹、随意第二人称「我们」搬进正文。  
- **勿**把他人署名作业中的经历写成「笔者」经历，除非事实仍成立且用户确认。  
- 事实、数据、图纸以用户**当次任务**材料为准；五篇仅作**语体与结构**参考，不作事实来源。
