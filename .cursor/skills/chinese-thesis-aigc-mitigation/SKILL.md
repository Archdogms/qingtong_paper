# 中文论文：降低「AI 写作检测」（AIGC 疑似率）

## 术语对齐（避免误解）

- 用户口语里的「**查重**」若指：**检测报告里的 AI 生成占比 / AI 疑似率 / AIGC 率**（各平台命名不同），即**判断文字是否像 AI 写的**那一项——**本 skill 专指这一类**，全文按 **AIGC 降信号** 处理。
- **另一类**：与期刊、学位论文、网络文库比对得到的**文字复制比 / 重复率**（传统文献查重）。只有用户**明确**要处理这一类时，才额外强调规范引用与转述；**不要**把两种报告混为一谈。

## 合规前提（必须先遵守）

- 遵守院校对**生成式 AI 使用与学术诚信**的规定；如需声明，协助撰写**真实可核对**的说明。
- **禁止**编造文献、数据、访谈与案例。改写只能基于用户提供的真实材料与可查来源。
- 目标：让文本呈现**可核对的人类研究痕迹**（具体对象、推理链、个人判断边界），而不是教唆伪造或欺骗性陈述。

## 语体底线：书面化、严谨性优先

- **降 AIGC 不能以「口水化」为代价**。输出须是**现代汉语书面语**：术语规范、句子完整、逻辑显式；避免讲稿腔、网络口语、过多感叹与叠词、随意插科打诨式转折。  
- **严谨性**：判断须有依据（文献、规范、场地条件、图纸或数据）；避免空泛褒贬与「显然」「不言而喻」式省略论证——若用「可见」「据此」等收束，前面须有可指代的具体内容。  
- **参考 `skill参考论文` 五篇时**：主要借鉴**篇章结构、分点论证、案例与数据的嵌入方式**，**不要**把其中口语连接词、演讲体、中英夹杂 pre 腔**原样搬进**学位论文或设计说明正文；`pre讲稿.docx` 仅作口头报告参考，**论文学语体须更正式**。  
- 在此底线之上，再处理模板句、对称排比、万能段等 **AIGC 高风险形式**。

### 重要提醒：书面严谨 ≠ 堆砌「标准学术套语」

部分检测器对「**高度整齐、高度通用、缺少课题毛刺**」的文本给分偏高——而这恰恰是许多大模型写「规范论文」时的默认输出。若改写后全文充满**可迁移到任意题目的书面连接词**（如反复「旨在、据此、综上所述、本研究、一方面另一方面」）却缺少**图号、数据、场地限定、非常规具体判断**，可能出现 **AIGC 率不降反升**。  
**对策**：书面语保留，但**必须**与「强化策略」中的**材料锚定、段首多样化、文献张力、克制模态**同时落实；宁可保留少量**与课题强绑定**的不齐整具体句，也不要写成「光滑的标准范文」。

## 何时应用

用户拿到 **AIGC / AI 写作检测报告**，或说「**别被检测成 AI 写的**」「**降 AI 率**」「**AI 查重**」等，对**中文学位论文或毕业设计说明**做逐段改写时使用。

## 同一稿件多次套用本 skill 的风险（必读）

有用户反馈：**连续两次**在已基本定稿的正文上，再按本 skill 做「全文式降 AIGC 改写」，**AIGC 疑似率反而大幅上升**。可理解机制包括：

- **文风收敛**：第二轮容易在已书面化的句子上再叠**同一批衔接词、元话语、段尾模板**（如反复「据此」「综上所述」「旨在」），全文越来越像**单一模型输出的统一腔**，检测器对该类统计特征敏感。  
- **信息不增**：若没有新的图号、数据、现场条件或文献分歧进入稿面，多轮「润色」只是在**重排套话**，对「像不像人写的」帮助有限甚至为负。

**硬性建议（Agent 须遵守）**：

1. **同一版本、同一章节的正文**：在本 skill 指导下做**结构化改写最多一轮**。用户若未提供**新的事实材料**，**不得**对全文发起第二轮「同指令再降一遍」。  
2. **若用户明确说「已经用本 skill 改过一次」**：**禁止**对同段再做全盘套用；仅允许：**(a)** 按检测报告**局部**改标红句；**(b)** 在用户提供**新锚点**（数据、图号、规范条、调研记录）后做**增写**而非重写。  
3. **二轮唯一正当理由**：用户补充了新材料或检测范围/平台变更，需要**把新信息写进论证链**——此时优先**加句、加段、加互证**，而不是把已有句子再抛光一遍。

## 工作流程（针对 AIGC 检测）

**（可选）**若 [`author-voice-sample.md`](author-voice-sample.md) 已含用户粘贴的真实节选，**改写前先读一遍**，对齐节律、术语与落点方式；规则见下节「用笔调样本校准」。

1. **定段落功能**：事实 / 概念 / 综述 / 方法 / 方案分析 / 总结。功能空洞（满段价值判断、无对象）最易被判模型生成。
2. **统一书面语体**：同一篇内术语、人称（如「本文」「该项目」）前后一致；书面衔接词**轮换使用**，**禁止**在同一段或相邻段重复堆砌「据此」「综上所述」「旨在」「从上述分析可知」；**节制**口语标记。若稿件**已按本 skill 改过一轮**，本步以**删重复衔接、保留差异句**为主，不再批量植入新的元话语。
3. **材料锚定（最关键）**：每段至少一处**硬锚点**——项目地名、建筑面积与层高、具体节点、材料与构造、规范号、图号表号、调研或实测数据（**必须来自用户材料**）。缺信息就向用户要，**禁止**编造数字装“像人写的”。
4. **打碎模板**：删除换任意题目都能用的句群；段首避免「随着/近年来/在…背景下」万能起势；少用对称排比当论证。
5. **把判断写“可检验”**：用「在某条件下会导致…」替代空形容词；方案句尽量 **图—文互锁**（见图 X、对应表 X）。
6. **节律与段尾**：避免每段同一位置「此外/因此/综上所述」；收束方式可交替，但须保持**陈述克制**，避免情绪化总结。
7. **引注**：若段落实质为教材/百科/他人原文，先规范**引用或转述+文献**——这既关学术规范，也避免大段“公共语料腔”叠加模型腔。
8. **段落级二次通读**：连读相邻 2～3 段，检查段首主语、连接词、段尾收束是否**同型重复**；若重复则只改其中一段的结构（信息顺序或句式骨架），保持书面语不变。

## 强化策略：在书面语下进一步压低 AIGC 信号

在已满足「语体底线」与「材料锚定」的前提下，优先使用下列手段（**不靠口语、不靠乱码**）：

1. **段首多样化（书面范围内）**：避免连续多段以同一词起笔（如句句「本文」「该空间」）。可轮换：**条件从句起笔**（「在柱网为 ×× 的前提下……」）、**图纸起笔**（「图 X 所示平面关系表明……」）、**规范或文献起笔**（「依据 JGJ 64—2017 对就餐区与后勤关系的原则性要求……」）。  
2. **信息梯度**：单段内尽量「**具体材料 → 推断或归纳 → 与图号/表号互锁**」，避免「观点—形容词—口号」空转。  
3. **综述写出张力**：对分歧文献用「**尚无一致结论**」「**存在不同解释路径**」等**书面**表述，再写明本文取舍理由；避免全篇「均认为」「一致认为」式平滑列举。  
4. **克制模态与边界**：设计说明多用「**宜**」「**应结合审图意见**」「**方案阶段以……为原则**」等，既严谨又区别于模型惯用的绝对断言。  
5. **压缩模型偏爱节律**：减少「**一方面…另一方面…第三…**」机械三分、连续「**致力于 / 旨在 / 充分**」类空动词堆叠；若需分点，用**编号 + 每条一句完整因果**优于排比形容词。  
6. **标点与版面**：少用非常规标点组合充当节奏；引号用于术语首次界定或直接引用，避免装饰性引号。

## AIGC 高风险句式 → 改写方向

| 常见模式 | 为何像 AI | 改写方向 |
|----------|-----------|----------|
| 段首「随着…近年来…背景下」 | 训练语料高频模板 | 直接写研究对象 + 矛盾或方法 |
| 「不仅…而且…」「既…又…」对仗论证 | 模型偏爱对称 | 拆成因果短句；插入条件、数据、图号 |
| 「提升体验/满足多元需求」等 | 万能句、无锚点 | 落到具体人群行为与空间节点（动线、照度、噪声、家具尺度） |
| 综述机械「A 指出…B 认为…」 | 句式重复、像生成列举 | 按**争议或问题**聚类，写分歧与你为何取这一支 |
| 破折号、冒号堆叠做“金句感” | 常见模型排版习惯 | 按中文标点收敛，多用句号切分 |
| 全文连接词位置整齐划一 | 节律机械 | 用**问题递进**衔接小节，少用套话转折 |
| 段段以「本文旨在/本设计致力于」起句 | 模型常用元话语 | 改为**条件、图纸或规范**起笔，元话语偶尔点题即可 |
| 「一方面…另一方面…第三…」机械三分 | 结构像生成列举 | 改为有主次的**两条因果链**，第三条升格为小节末「局限」 |
| 连续「重要、核心、关键」等评价堆叠 | 无信息增量 | 改为**可度量对象**或删留一处 |
| 定义段大段无引号的他人表述 | 百科腔 + 易判 AI | **短引+文献**或**转述+文献**，并接本项目用法一句 |

## 结构与密度（设计类）

若仓库有 `论文/规范依据_OCR摘录.md`，在合规章节里填入**仅本项目**的观察与对策（前言—概况与分析—构思—方案—总结），比单纯换词更能拉低 AI 疑似度。

## 交付格式

1. 若存在大段未标注的他人原文：先说明须**补引注或转述**，不替用户掩盖抄袭。
2. 给出 **3～7 条**修改要点，**明确针对 AIGC/AI 检测**（勿再写“文献查重”除非用户提了）。
3. 给出改写后段落；括号标注需用户补充的数据、图号、文献页码。

## 自检清单（AIGC 向）

- [ ] 全文语体是否为**书面语**，无讲稿腔、网络腔与多余口语赘语？
- [ ] 每段是否至少有一处**离开本项目就不成立**的具体信息？
- [ ] 是否删除了可套到任意题目的句群？
- [ ] 方案与分析是否尽量挂了**图号/表号/可核对条件**？
- [ ] 连接词与段尾收束是否避免机械重复，且衔接**克制、可复核**？
- [ ] **相邻段**段首是否避免同一主语/同一连接词模板连打？
- [ ] 是否减少「旨在/致力于/充分」与「一方面另一方面第三」等**模型高频骨架**？
- [ ] 综述与定义是否出现**文献张力或限定条件**，而非全肯定平滑叙述？
- [ ] 学校对 AI 使用披露、图表与参考文献要求是否已提醒用户核对？
